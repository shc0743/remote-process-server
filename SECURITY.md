# Security Policy

## Supported Versions

Currently supported versions with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 3.1.x   | :white_check_mark: |
| 3.0.x   | :x:                |
| 2.x.x   | :x:                |

**Note**: Pre-release versions (alpha, beta, rc), temporary versions (hotfix), and internal versions are **not** supported for security updates. Please always use the latest stable version.

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

To report a vulnerability, please use GitHub's **Security Advisory** feature:

1. Go to the repository: https://github.com/shc0743/remote-process-server
2. Click the **Security** tab
3. Click **Advisories** in the left sidebar
4. Click **New draft security advisory** (green button)

**What to expect:**

- You will receive acknowledgment of your report
- We will provide updates on the status
- Once the vulnerability is verified, we will:
  - Work on a fix in a private temporary fork
  - Credit you in the published advisory (unless you prefer anonymity)
  - Release a patched version
- If the report is declined (e.g., not a security issue, out of scope), we will explain the reason

**What to include in your report:**

- A clear description of the vulnerability
- Steps to reproduce the issue
- Affected versions
- Potential impact
- Any suggested fixes (optional)

Thank you for helping keep `remote-process-server` secure!
